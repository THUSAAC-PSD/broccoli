# 打印

参赛者在网页界面发起打印自己代码的请求。志愿者监管一个实时队列，打印机一侧的机器上
运行的小程序则将每个请求渲染为整洁、带语法高亮的 PDF 并打印出来。

## 工作原理

打印功能由三部分组成。

1. **插件**，位于 Broccoli 服务端内部。它维护队列并决定由谁打印何种内容，自身从不与
   打印机通信。
2. **网页界面**，即参赛者与志愿者日常使用的界面。参赛者在其中发起打印请求，志愿者在其中
   操作队列。
3. **打印站**。打印站是一台放置在打印机旁的普通计算机，运行一个名为 `print-client` 的
   小程序，负责实际的渲染与打印。

服务端与打印站被有意分离。服务端运行在只能返回数据、而不能返回 PDF 或打印机指令的沙箱
中，因此产出纸质件的环节必须运行在打印机旁、由你掌控的机器上。这种分离还意味着，即使
打印站短暂断网，打印也不会中断，任务会留在队列中，直到打印站重新领取。

一个打印站可以驱动多台打印机，也可以同时服务多个比赛服务端。

## 启用打印

打印在打印插件的设置中配置。在管理区打开 **插件**，找到打印插件，然后打开其 **配置**
对话框。下列设置既可对所有比赛全局生效，也可仅对某一个比赛生效。

| 设置项               | 作用                                                                                                   |
| -------------------- | ------------------------------------------------------------------------------------------------------ |
| **启用打印**         | 总开关，默认关闭。                                                                                     |
| **需要批准**         | 开启后，每个请求都会停留在 _等待批准_ 状态，直至志愿者批准；关闭则请求直接进入打印流程。               |
| **每个任务的最大页数** | 预估页数超过此上限的请求，会在参赛者提交时即被拒绝并给出明确提示，可防止误打印数百页。                 |
| **页眉横幅**         | 打印在每页顶部的可选文字，通常为比赛名称。                                                             |
| **打印站令牌**       | 允许打印站接入的共享密钥，详见下方[打印站令牌](#打印站令牌)。                                           |

### 打印站令牌

打印站令牌是一个口令，允许某个打印站领取并打印某比赛的任务。其取值由你自行设定，任何
足够长且难以猜测的字符串均可，例如口令生成器的输出。将其加入 **打印站令牌** 列表，并把
相同的取值配置到你信任的每个打印站。

令牌分为两种。

- **比赛级令牌** 在某比赛的打印一节中配置，仅对该比赛生效。
- **全局令牌** 在插件的部署级设置中配置，对服务端上的所有比赛生效。当一个打印台服务
  整场赛事时，这种方式较为便利。

:::warning[请像对待密码一样对待令牌]

任何持有令牌的人都能取得参赛者提交打印的代码。请通过可信渠道分发令牌，并在不同赛事
之间予以更换。

:::

## 设置打印站

每台承担打印的机器在比赛开始前各设置一次。

你需要一台位于打印机旁的计算机，且打印机已在操作系统中安装并可正常工作，也就是你已经
能够从该系统正常打印一页普通测试页。此外还需要比赛服务端的地址和一个打印站令牌。

### 获取程序

从项目的 GitHub Releases 页面下载对应操作系统的 `broccoli-print-client`。它是一个自带
全部依赖的单文件程序，无需安装。

- 在 **Windows** 上，把 `.exe` 放在易于找到的位置。它需要在终端中运行，双击并无作用。
- 在 **macOS** 与 **Linux** 上，首次使用前需赋予可执行权限。在下载目录中打开终端，运行
  `chmod +x broccoli-print-client`，随后以 `./broccoli-print-client` 运行。

为简洁起见，下文统一写作 `print-client`，请替换为你实际下载的文件名。

### 运行设置向导

```bash
print-client setup
```

向导会找到本机已安装的打印机，询问服务端地址与打印站令牌，让你为打印站及其打印机命名，
并在程序旁写入一个名为 `print-client.toml` 的配置文件。多数场景下，这便是全部所需配置。

### 配置文件

`print-client.toml` 是可直接打开编辑的纯文本文件。下面是一个典型示例。注意，单个打印站
可以列出多个服务端与多台打印机。

```toml
station  = "room-a-desk-1"   # 自定义名称，会显示在志愿者面板中
location = "Room A"          # 可选；用于将任务路由到指定房间
poll_interval_secs = 3       # 向服务端轮询新任务的间隔
max_pages = 10               # 打印站侧同样强制执行的安全上限
paper = "A4"                 # 或 "Letter"
font_size = 9.0
banner = "Regionals 2026"    # 打印在每页顶部

[[server]]
url   = "http://judge.local:3000"
token = "你配置的打印站令牌"

[[printer]]
name  = "main"                         # 该打印机的标签
os_id = "HP_LaserJet_Room_A"           # 该打印机在操作系统中的名称
```

`os_id` 是操作系统所识别的打印机名称。在 Windows 上可于“设置”中依次进入“蓝牙和其他
设备”“打印机和扫描仪”查看；在 macOS 与 Linux 上可在终端运行 `lpstat -p` 列出打印机
名称。设置向导会自动填写此项，通常无需手动查找。

### 比赛前检查

```bash
print-client doctor
```

`doctor` 会确认各服务端可达且令牌被接受，列出检测到的打印机，并可向每台打印机发送一页
真实测试页。设置完成后运行一次，若测试页正常打印，即表示就绪。

### 比赛当天运行

```bash
print-client run
```

在整场比赛期间保持其运行。它会轮询服务端、领取任务、渲染并打印，同时逐条记录日志。
若机器重启，重新启动即可，尚未打印的任务仍保留在队列中。

:::tip[无需打印机即可试用]

将某个打印机条目指向一个文件夹而非真实硬件，即可预览输出，或在打印机到位前演练整个
流程。

```toml
[[printer]]
name    = "preview"
command = "folder:/Users/you/Desktop/prints"   # PDF 将写入此处，而非打印
```

也可以直接渲染本地文件。

```bash
print-client test-print solution.cpp --out preview.pdf
```

:::

## 任务如何送达打印机

打印站提供多种将 PDF 送往打印机的方式，因此无需任何额外软件即可适配不同操作系统与特殊
环境，也无需第三方 PDF 阅读器。

- 在 **macOS** 与 **Linux** 上使用系统内置打印（CUPS）。只要打印机能从操作系统打印，
  即可从此处打印。
- 在 **Windows** 上使用系统内置的打印处理程序。同样，只要 Windows 能打印，打印站即可
  打印。
- 对于标签打印机、原始队列等特殊情形，可用 `{printer}` 与 `{file}` 占位符提供自定义
  命令，例如 `command = "lp -d {printer} {file}"`。
- 设置 `command = "folder:/路径"` 会将 PDF 写入文件夹而非打印，适合测试、归档或完全手工
  的流程。

## 参赛者看到什么

启用打印后，参赛者会在其提交结果页看到 **打印** 按钮，并在代码编辑器附近看到一个打印
入口，用于粘贴或上传任意代码。发起打印后，参赛者会看到其状态（排队中、打印中、已打印），
从而判断是否需要前往打印台领取。若该比赛要求批准，请求会显示为等待批准，直到志愿者
放行。

## 操作打印队列

志愿者可使用 **打印队列** 面板。它列出每个任务的参赛者、题目、文件、页数与实时状态，
并随任务推进自动刷新。在此面板中，志愿者可以执行以下操作。

- **批准** 处于等待批准状态的任务。
- **重新打印** 输出有误的任务，使其重新进入队列。
- **取消** 不应打印的任务。
- 将一个或多个任务**指定**到特定打印机，便于把某房间的打印件送往该房间的打印机。
- 在放行前**查看**即将打印的代码。

另有一个独立面板显示各打印站及其是否在线，便于在参赛者前来询问之前，及早发现掉线的
打印站。

## 任务生命周期

![任务依次经历 pending_approval、pending、claimed、printing、done；可能失败或被取消，已结束的任务可重新打印。](/img/print-lifecycle.zh.svg)

新任务初始为 `pending`；若该比赛要求批准，则初始为 `pending_approval`，经志愿者批准后
转为 `pending`。随后某个打印站领取该任务、打印它，并标记为 `done`。渲染或打印出错会使其
转为 `failed`。志愿者可在任务完成前取消之，任何已结束的任务都可重新打印，从而重新回到
队列。

当多个打印站同时运行时，服务端通过原子领取将每个任务恰好交给其中一个打印站，因此即便
两个打印站在同一瞬间请求任务，同一任务也绝不会被打印两次。

## 语言与字体

打印内容远不止支持英文。打印站内嵌一款等宽字体，覆盖中文、日文、韩文，以及希腊文、
西里尔文与带重音的拉丁文，并以正确的等宽对齐与逐词法单元的语法高亮进行排版。字体所缺的
字符（如表情符号）会显示为占位方框，而不会直接消失。

## 故障排查

| 现象                       | 可能原因与解决办法                                                                                |
| -------------------------- | ------------------------------------------------------------------------------------------------- |
| `doctor` 无法连接服务端    | 服务端地址有误，或机器不在比赛网络内。请检查 `print-client.toml` 中的 `url`。                      |
| `doctor` 提示令牌被拒绝    | 令牌不在该比赛（或全局）的 **打印站令牌** 列表中，或存在笔误。请重新复制。                         |
| 找不到某台打印机           | `os_id` 与操作系统中的打印机名称不一致。请运行 `lpstat -p`（macOS、Linux），或在“打印机和扫描仪”中核对（Windows）。 |
| 任务停留在 _等待批准_      | **需要批准** 已开启。请在打印队列中批准，或关闭该设置。                                            |
| 请求因过长被拒绝           | 超过 **每个任务的最大页数**。请提高上限，或请参赛者精简文件。                                      |
| 非拉丁字符显示异常         | 通常因内嵌字体而正常。若出现占位方框，说明字体缺少该字形，例如表情符号。                           |
